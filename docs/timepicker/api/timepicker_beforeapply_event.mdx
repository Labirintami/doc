<Short>
 fires before saving the timepicker value



</Short>



- value		string|object	the value of a timepicker	


-result		boolean		false - to prevent saving the timepicker value, otherwise true

~~~js

timepicker.events.on("BeforeApply", function(value) {
    console.log(value);
    return false;
});


~~~
 api_event

<Descr>

The type of the value parameter depends on the applied [valueFormat](/timepicker/api/timepicker_valueformat_config) and [timeFormat](/timepicker/api/timepicker_timeformat_config):

- If *valueFormat: "string"*  and *timeFormat:24* are specified, the string value will include just the hour and minutes: "00:39"
- If *valueFormat: "string"*  and *timeFormat:12* are specified, the string value will include hour, minutes, and am/pm identifiers: "06:00AM"
- If *valueFormat: "timeObject"*  and *timeFormat:24* are specified, the object value will contain key:value pairs for hours, minutes and their values: {hour: 0, minute: 39}
- If *valueFormat: "timeObject"*  and *timeFormat:12* are specified, the object value will contain key:value pairs for hours, minutes, am/pm identifiers and their values: {hour: 6, minute: 0, AM: true}


</Descr>
